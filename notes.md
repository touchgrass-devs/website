  - **Lanyard icon now syncs to the active philosophy pillar:** the fixed `>_` mark is gone from the front face - `LanyardBadge` now receives `activeIndex` (lifted into state in `Philosophy.jsx`, set from the same scroll `onUpdate` that drives the card stack) and shows whichever pillar's own icon is currently on top of the stack: hammer (Craft), gauge (Performance), coins (Pricing) - real Phosphor glyphs (fetched from `phosphor-icons/core` on GitHub, rasterized via ImageMagick/librsvg, recolored to the brand green programmatically since ImageMagick doesn't resolve SVG `currentColor` fills) composited into three front-face PNGs (`card-front-craft.png` / `-performance.png` / `-pricing.png`). All three are preloaded together in one batched `useLoader` call so swapping `activeIndex` just switches between already-decoded textures, no pop-in.
  - **Glow/lighting fully removed, on repeat feedback that it "looked odd":** went through a pointLight (created a specular hotspot sliding around as the card swayed on its rope), then an additive glow-sprite plane (still didn't read right) - both are gone now. Card face is deliberately plain: `card.glb`'s mesh now uses `meshStandardMaterial` at `roughness: 0.95, metalness: 0.05` (no more `meshPhysicalMaterial`/clearcoat), and the scene's light rig is down to one `ambientLight` + one plain `directionalLight` - no point light, no color tint, no HDRI. `glow-sprite.png` is unused now (left in `public/lanyard/`, harmless).

Card background is now flat (`card-front-*.png` no longer carries the baked glow-halo gradient behind the icon, since the whole point of that gradient was the now-removed glow effect).
