## Hero section (built)
- **Divergence from blueprint (background):** blueprint's static CSS grid + hardcoded-SVG "hands" is fully replaced by `components/HeroDotGrid.jsx` — a canvas that samples `public/hero/creation-hands.jpeg` (max-pooled per grid cell, so sparse dark ink survives downsampling) into an interactive halftone dot field. Dots are visible at rest in their sampled grayscale; pointer proximity shifts them toward `grass-accent` green and scales them up, proportional to distance and to each dot's own ink (so background/edge noise doesn't react). Resting frame is cached to an offscreen canvas and blitted per frame; only dots near the pointer are individually redrawn — needed after the first version was laggy at full dot-count redraw. This is the new source of truth for the hero background — don't revert toward the blueprint's static version.
- `components/Hero.jsx`: blueprint's layout/structure (frame, spacing, motion stagger) with the new background. Headline emphasis uses same-family bold italic, not a serif swap (was Playfair Display - fixed per `design-taste-frontend`'s ban on serif-into-sans-headline emphasis).
- Copy is sourced from `aboutus.md` (project imported knowledge - 4-person team, web design/dev, apps, AI integration, maintenance) blended with blueprint phrasing where it already fit. First draft, not final: headline "Clean Engineering, / Modern Design", CTA "Start a Project", footnotes kept close to blueprint tone plus a "Touch grass." pun.

## Services section (built, revised)
- **Divergence from blueprint (full layout swap):** blueprint's `Services.tsx` (left menu list + right detail card, tab-style) is fully replaced by a 3D coverflow carousel, redesigned per a reference video the user supplied. This is the new source of truth for this section's structure - don't revert toward the blueprint's split-selector layout.
- `components/Services.jsx`: section header (eyebrow "Our Services" + heading + short paragraph, blueprint's header-row composition) sits above the carousel - reinstated on user request after an eyebrow-restraint pass had dropped it; this is the one eyebrow on the page, within budget.
- Carousel is a true circular "drum": `ringPlacement(offset)` uses a **linear** angle (`offset * STEP_DEG`, constant angular velocity) - a prior version used an accelerating `offset^2` curve to force the opposite card to a literal 180deg, but that made drag-to-rotate feel disconnected from the finger (non-constant angular speed doesn't feel like spinning a real ring). Reverted to linear; the opposite card (offset 2, the only other one when N=4) now lands at `2*STEP_DEG` (`STEP_DEG=62`) - not literally hidden behind, but small/faint/low z-index enough to sit mostly inside the active card's footprint. `RADIUS=260`, container `perspective: 900px` (dialed down from 1300 + a much larger radius) so side cards actually show visible keystone/bend, per a reference screenshot - the previous combination was too mild to read as "curved."
- The drag-to-rotate mechanism (`spinSteps`, pointer handlers, velocity-seeded spring settle) is unchanged in structure - only the angle formula and constants moved, so the "spins fast then locks" feel should now track the drag directly instead of via an eased/accelerated mapping.
- Carousel is swipe-only: pointer handlers (attached per-card, with `window` move/up listeners while a drag is active - not relying on the native `click` event, which was unreliable for tap-to-select) track drag distance into `spinSteps` (a Framer `useMotionValue`) for live rotation feedback; on release, a non-moved pointer selects that card directly, a moved one computes release velocity and spring-settles `spinSteps` back to 0 with that velocity seeded in (fast flicks spin further before locking). No autoplay, no play/pause/next transport buttons - replaced with a minimal dash-style step indicator below the carousel.
- **Card color scheme corrected:** first pass used a flat near-black (`sage-950`) card face, which read as a generic "AI dark-mode tile" clashing with the rest of the light, luxury palette - fixed per feedback. Cards are now `bg-white` with `luxury-border`, matching the blueprint's actual card treatment; identity per card comes from icon + a very faint `grass-accent-light` positional wash (kept faint to respect the one-accent-color rule), not from a dark base.
- **Hover glow:** each card now has a mouse-tracking radial glow plus a gradient-following border ring (via the `mask`/`WebkitMaskComposite: exclude` border-gradient technique), reusing the exact accent RGB (`63, 174, 106`) from `HeroDotGrid`'s hover color for brand continuity, per a reference card screenshot the user supplied (that reference used a purple/pink scheme - adapted to our locked green accent instead of copying its hue).
- Only the focused card expands: icon + divider + a vertical list of that category's sub-services, each with a short one-line explanation (sourced from the blueprint's `data.ts` sub-service descriptions, condensed to fit) - a first pass had only bare labels with no explanation text, which lost content the blueprint had. Unfocused cards show icon only, with a hover state (lift, icon tint, title label fade-in). Full title + description for the active category stays below the carousel.
- Installed `@phosphor-icons/react` (not previously a dependency) - `design-taste-frontend` discourages `lucide-react`.
- **Perceived animation lag fixed:** slow 300ms defaults were the main cause - shortened hover/tap transitions across the section to 150-200ms and gave `whileHover` its own fast tween instead of inheriting the card's positional spring (a spring starting from zero velocity reads as a slow ramp-up for hover feedback).
- **Caption/card-content crossfade bug:** removing `AnimatePresence mode="wait"` (to fix the above) caused a *different* problem - old and new text visibly overlapped mid-transition, unreadable. Re-added `mode="wait"` on both the card content swap and the caption, but with short exit durations (~100-120ms) so the "wait" is barely perceptible while still guaranteeing only one text block is ever visible at a time.
- **Navigation replaced again:** dash-style step indicator swapped for two liquid-glass arrow-bubble buttons (prev/next) - translucent `bg-white/40` + `backdrop-blur-xl` + inner highlight border, per feedback that dashes weren't the right call either.
